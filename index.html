Neural Network Models

Description

The visualization shows how a neural network classifier works on a dataset. The visualization gives the user the opportunity to tweak the dataset as well as the network using different parameters (learning rate, batch size, number of hidden layers and features, dataset type, etc). When all parameters are set (either manually by the user or automatically using the default settings), the user can press a button which starts the neural network learning simulation in which he will be able to see how the classification on the data set progresses with time up until the point when no more changes are made to the classes assigned to the points in the dataset.

Positives

The high number of parameters that the user is able to set makes it possible to simulate more different kinds of situations. This helps to better understand how neural networks work, because the user is able to see how they work on different kinds of data with different settings.

Further, the two colors that are used to classify the data into two groups are very different (blue and orange) which makes the classes easily distinguishable. Also, the colors are used with different intensities indicating how confident a certain prediction is. This increases the information that the visualization conveys even more.

Negatives

Even though the high number of variables can be seen as a good thing, it can also be a problem as the visualization may seem overwhelming or confusing for a user. Especially for users that are not very familiar with neural networks, the possiblity to tweak so many parameters may result in even more confusion about how neural networks work. However, the text underneath the visualization helps understand how the visualization works and what the different parameters mean. But that is less clear from the visualization itself.

Another problem with the visualization is how the hidden layers are visualized. The more layers that are added to the network, the lower the intensity of the classification colors is. Therefore, despite the fact that this showcases the unclarity of the underlying mechanisms in a neural network, it becomes quite unclear what information the low intensity layers convey.

Map of Danish Dialects

Description

The map visualizes the distribution of Danish dialects with respect to numerous different linguistic features (e.g. where do people say Ã¦ 'I' instead of jeg 'I'). The user is able to choose one of the several layers of the map to view the distribution of these features. Further, the user is at every layer allowed to hear examples of both young and old speakers from a specific region by clicking on coloured dots pinned to the map.

Positives

The map visualizes the distribution of the dialects with both colors and patterns, which enables the user to deduce similarity/overlap between different features easily. Having the dialect examples pinned to the map allows the user to 'dive' into the regional differences where more exhaustive and detailed information is available. In this way the user is able to keep an overview of the overall tendencies while exploring single data points qualitively.

Negatives

The map contains a lot of information and even though it makes use of different features to try to partion the data into different levels of information detail, a great overview is still missing. The map is good for browsing and discovering fun features, but it is not fit for surveying regional differences. Being able to select different feature layers on top of each other (e.g. by weightening with color saturation) would definitely help to compare the different features and their distribution on the regions.

Sidsel

I think graphs would be an interesting data structure to try and visualize. Graphs vary greately with respect to nubmer of data points, how different nodes are connected and how different connections are weighted and what semantics those carry.

Topology is an example of what a graph can express and I could imagine doing a project on that. Topology can both be viewed as a 'local' feature between two elements but might also be viewed 'globally' comparing the different kinds of relations between data points.

An idea for a project could be to visualize names or word and where they come from. A classic example of how to visualize such a structure is by creating a tree with roots and brances. It would be cool to create a model to create such trees generically, choosing e.g. unique relations (e.g. Danish name or German name) or tags (e.g. biblical name) as branches.

Wanting to work with words one could try to scrape or aquire a dump from Wiktionary, which annotate connections between different words.

To enable the viewer to get an overview over the dataset or different parts of the dataset would be a goal, so that the different relations of the dataset could be explored.

Mai-Britt

Very much in line with the neural network visualization above, I would like to be able to make a visualization that could show how different classification algorithms (k-nearest neighbor, logistic regression etc) work on data. The data would be randomly generated using different parameter settings (like type and size of data, mean and variance) that can be manually set by a user. The algorithm settings can be tweaked as well (like the value of k in k-nearest neighbor). This will help to show how the algorithms work with many different kinds of data and not just one or two types as is most often the case when you search online for how the algorithms work.

The visualization would be accompanied with some text explaining the most important characteristics of the algorithm in question, if possible in bullet points to make the information conveyed as clear as it can be.

The audience would be high school or university students that take courses in machine learning, data science or the likes and other people who are interested in these topics.
